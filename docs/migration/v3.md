# Migrating to v3.0.0

This guide covers upgrading projects from `atom-websdk@2.0.4` to `v3.0.0`.

## CDN URL

```html
<!-- Before -->
<script src="https://cdn.jsdelivr.net/gh/FootLooseLabs/atom-websdk@2.0.4/dist/sdk.min.js"></script>

<!-- After -->
<script src="https://cdn.jsdelivr.net/gh/FootLooseLabs/atom-websdk@3.0.0/dist/sdk.min.js"></script>
```

---

## Breaking changes

### 1. `Muffin.Service` is now built-in

`Service` is a first-class export from `atom-websdk`. Remove your local polyfill â€” it is no longer needed.

**wity-agent-builder** â€” remove from `4_polyfills.js`:
```js
// DELETE â€” ElementWebService class and everything below it in 4_polyfills.js
Muffin.Service = class ElementWebService { ... }
```

**jity-dam / wity-accounts-widget** â€” remove the file and its import:
```ts
// DELETE this file entirely
src/muffin-sdk/service-polyfill.ts

// DELETE this import from src/muffin-sdk/index.ts or wherever it's imported
import { applyServicePolyfill } from './service-polyfill'
applyServicePolyfill()
```

Your service classes need no changes â€” they extend `Muffin.Service` exactly as before.

---

### 2. `uiVars` is now a reactive Proxy

Setting any `uiVars` property automatically schedules a render. Manual `render()` calls after `uiVars` mutations are now redundant (but still safe â€” they flush immediately).

**No code changes required.** Existing `this.render()` calls continue to work. You may remove them incrementally as you touch files.

```js
// Before â€” required
this.uiVars.loading = false
this.render()

// After â€” render() is optional, uiVars mutation schedules it automatically
this.uiVars.loading = false
```

Multiple mutations in the same call produce one render (batched via `queueMicrotask`).

---

### 3. DOM helper methods are now built-in

All `DOMComponent.prototype` extensions from `4_polyfills.js` are now applied automatically by `atom-websdk` on load. Remove them from your polyfill file.

Methods to remove from `4_polyfills.js`:

```js
// DELETE all of these â€” now built into atom-websdk
Muffin.DOMComponent.prototype.getElement = ...
Muffin.DOMComponent.prototype.getElements = ...
Muffin.DOMComponent.prototype.toggleBtnBusyState = ...
Muffin.DOMComponent.prototype.toggleRootAttr = ...
Muffin.DOMComponent.prototype.toggleSurface = ...
Muffin.DOMComponent.prototype.isSurfaceActive = ...
Muffin.DOMComponent.prototype.toggleTargetSurface = ...
Muffin.DOMComponent.prototype.toggleTargetTab = ...
Muffin.DOMComponent.prototype.awaitChildLoad = ...
Muffin.DOMComponent.prototype.initSubscriptions = ...
Muffin.DOMComponent.prototype.notifyUser = ...
Muffin.DOMComponent.prototype.copyToClipboard = ...
Muffin.DOMComponent.prototype.callParent = ...
Muffin.DOMComponent.prototype.callGrandParent = ...
Muffin.DOMComponent.prototype.getSessionUser = ...
Muffin.Router.prototype.updateHistory = ...
```

---

### 4. Global utility extensions are now built-in

Remove from `4_polyfills.js`:

```js
// DELETE â€” now applied automatically
Array.prototype.splitIntoMultipleArrays = ...
String.prototype.ellipsify = ...
```

---

### 5. `window.Muffin` is the single namespace

Bare globals (`window.DOMComponent`, `window.PostOffice`, `window.DataSource`) have been removed. Use `Muffin.DOMComponent`, `Muffin.PostOffice` etc.

Check your codebase for any direct references:

```bash
grep -r "window\.DOMComponent\|window\.PostOffice\|window\.DataSource" src/
```

## New in v3.0.0

### `Muffin.createStore()`

Named reactive store with opt-in persistence and socket sync:

```js
const sessionStore = Muffin.createStore('session',
    { user: null },
    { persist: true }
)
```

See [Stores](/guide/stores) for full API.

### `derived` on components

Declarative computed state â€” no more manual boolean chains:

```js
static derived = {
    isEmpty: (uiVars, data) => uiVars.items.length === 0
}
```

### `esc()` in every `markupFunc`

Built-in HTML sanitization â€” available automatically, no import needed:

```js
static markupFunc(_data, uid, uiVars) {
    return `<p>${this.esc(uiVars.userInput)}</p>`
}
```
