Muffin Framework Test Rig

Test 1 — Basic render

Component should render with correct uiVars set in constructor.

Test 2 — uiVars reactivity

Clicking the button should increment the counter (re-render).

Test 3 — Inheritance (mirrors wity-app pattern)

Subclass extends base, both have markupFunc, subclass overrides.

Test 4 — uiVars set in constructor + onConnect

Constructor sets defaults; onConnect sets real values. Both should render.

Test 5 — stateSpace with idle root (mirrors wity-app pattern)

Component defines stateSpace without idle; idle merged via defaultStateSpace. onConnect calls switchState("loading") then switchState("loaded"). Should NOT show idle state after loaded. Should NOT cause extra renders.

Test 6 — switchToIdleState does NOT trigger extra render

Component with onConnect that has an await before switchState. switchToIdleState() should not schedule a stale render. Render count should be minimal (no spurious idle renders).
